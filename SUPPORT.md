# Support

GISPulse comes with two support tiers — community (free) and enterprise (paid).

## Community support (free, AGPL users)

- **GitHub Discussions** : https://github.com/imagodata/gispulse/discussions — Q&A, ideas, show & tell.
- **GitHub Issues** : https://github.com/imagodata/gispulse/issues — bug reports and feature requests only (please use the templates).
- **Stack Overflow** : tag your question with `gispulse`.

Best-effort response time. Maintainers triage on weekdays.

Before opening an issue :
1. Search existing issues / discussions.
2. Reproduce on the latest `gispulse` from PyPI (`pip install -U gispulse`).
3. Include a minimal reproducer, full error message, Python and OS versions.

## Commercial support (paid, GISPulse Enterprise customers)

If your organisation cannot use AGPL or needs SLAs, SSO, RBAC, billing,
or premium connectors, contact **sales@imagodata.com** for the
commercial dual-license + Enterprise plugin (`gispulse-enterprise`).

Enterprise customers get :
- 4-hour response SLA for P0 / P1 issues
- Direct Slack / email channel with maintainers
- Priority bug fixes and patch backports
- Architecture review sessions
- Optional self-host or managed cloud deployment

## Security issues

See [SECURITY.md](SECURITY.md). Do not open public issues for vulnerabilities.

## Operational status (cloud / demo)

- Demo : https://demo.gispulse.dev (best-effort, no SLA)
- Status : https://status.gispulse.dev (when applicable)
