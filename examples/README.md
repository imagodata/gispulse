# GISPulse Examples

This directory contains sample datasets and rule files to get started with GISPulse.

## Generating Sample Data

Run the generation script once from the project root to produce the GPKG files:

```bash
python examples/generate_samples.py
```

This creates:
- `examples/urban_planning.gpkg` — 2 layers: `parcels` (20 features) and `roads` (10 features)
- `examples/points_of_interest.gpkg` — 1 layer: `pois` (30 features, Paris area)

A second dataset (`city.gpkg`) with a French urban planning scenario is generated by:

```bash
python examples/create_sample_gpkg.py
```

This creates `examples/city.gpkg` with 3 layers: `parcels`, `flood_zones`, `schools`.

## Rule Files

### filter_zones.json — Filter by zone type

Keep only residential parcels from `urban_planning.gpkg`:

```bash
gispulse run examples/urban_planning.gpkg \
  -r examples/filter_zones.json \
  -o output/residential.gpkg \
  --layer parcels
```

### buffer_roads.json — Buffer roads by 10 m

Apply a 10 m buffer around all road geometries:

```bash
gispulse run examples/urban_planning.gpkg \
  -r examples/buffer_roads.json \
  -o output/roads_buffered.gpkg \
  --layer roads
```

### filter_pois.json — Keep only high-rated POIs

Filter points of interest with a rating >= 4.0:

```bash
gispulse run examples/points_of_interest.gpkg \
  -r examples/filter_pois.json \
  -o output/top_pois.gpkg \
  --layer pois
```

### pipeline_enrich.json — Multi-step pipeline

Filter large parcels (area > 1000 m2) then apply a 50 m buffer:

```bash
gispulse run examples/urban_planning.gpkg \
  -r examples/pipeline_enrich.json \
  -o output/enriched_parcels.gpkg \
  --layer parcels
```

### pipeline_full.json — Full urban pipeline (city.gpkg)

Filter commune-owned parcels, buffer 50 m, then dissolve:

```bash
gispulse run examples/city.gpkg \
  -r examples/pipeline_full.json \
  -o output/commune_zone.gpkg \
  --layer parcels
```

### cross_layer_intersect.json — Spatial filter against another layer

Keep only parcels that intersect the roads layer (both layers must be in the same GPKG):

```bash
gispulse run examples/urban_planning.gpkg \
  -r examples/cross_layer_intersect.json \
  -o output/parcels_near_roads.gpkg \
  --layer parcels
```

### cross_layer_intersects.json / cross_layer_clip.json / cross_layer_spatial_join.json

Additional cross-layer examples using `city.gpkg` (parcels vs flood_zones vs schools).
See each JSON file for the full rule configuration.
